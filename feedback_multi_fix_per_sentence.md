Применимо: wife-english-tutor, son-french-tutor (любой LLM-tutor с `[FIX]...[/FIX]` контрактом).

**Why:** 2026-05-11 жена пожаловалась что бот корректирует только первое из 2-3 ошибочных предложений + не понимает русский ("Sorry, what?") + отвечает 1-2 предложениями ("скудно"). Старый промпт явно говорил "Только ОДНА поправка за ход" — это и был баг.

**How to apply:**
1. **Multi-FIX в промпте.** Заменить "Только ОДНА поправка" на "НЕСКОЛЬКО ПРЕДЛОЖЕНИЙ С ОШИБКАМИ — НЕСКОЛЬКО [FIX]". Дать пример: "I go to store yesterday. I buyed milk." → `[FIX]Say: "I went to the store yesterday."[/FIX] [FIX]Say: "I bought milk."[/FIX] Then natural reply.`
2. **`split_reply()` через `_FIX_RE.finditer`** (не `.search`). Собрать ВСЕ [FIX], `re.sub` убирает теги из body, корrections склеиваются через `". "`. TTS произнесёт подряд.
3. **Русский input — абсолютное правило.** "ВСЕГДА понимай её русский. НИКОГДА не отвечай 'I don't understand'/'Could you repeat in English?' — это ЗАПРЕЩЕНО." Каждая русская фраза → свой [FIX] с переводом в target lang.
4. **Развёрнутый ответ.** Заменить "1-2 короткие реплики максимум 25 слов" → "3-5 предложений, развёрнутый ответ. Поделись реакцией, добавь деталь, задай уточняющий вопрос." `max_tokens` 400 → 700.
5. **Короткое сообщение = короткий ответ.** Добавить исключение: "Если её сообщение короткое и закрытое ('Goodnight') — твой ответ 1-2 предложения, но с теплом."

Файлы где правка применена 2026-05-11:
- `/root/wife-english-tutor/bot/llm.py` (PERSONA_TEMPLATE + split_reply + max_tokens)
- `/root/son-french-tutor/bot/llm.py` (тот же набор изменений на французском)

CELPIP-боты используют свой `bots/celpip_bot.py` с другим промптом — multi-FIX там не применён (CELPIP — экзамен-симулятор, у него своя логика).
