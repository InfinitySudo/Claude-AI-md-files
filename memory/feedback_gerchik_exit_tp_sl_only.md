**База метода Герчика по выходу из позиции = либо TP, либо SL. Никаких безубытков/трейлинга/лесенки по умолчанию.** Артём смотрел курс и настаивает; подтверждено сырыми транскриптами `/root/gerchik_course/*/*.txt`.

**Why:** математика метода — TP ≥3R (фикс. цель к следующему уровню). При WR~30% одна сделка +3R перекрывает 3 стопа; 3 тейка = +9R/день → слить депозит почти невозможно. БУ@1R это ЛОМАЕТ: подтягивает стоп в ноль на +1R и душит сделку до +3R («движение выдохлось» — куча mediocre-BE в журнале).

**Источник (БУ/трейл/лесенка ЕСТЬ в курсе, но как ПРОДВИНУТОЕ, не база):** `[14f9b1ab]` «тейки не менее 3 к 1… ДАЛЬШЕ когда будете работать можете дробить позу»; `[01251b30]` 10-10-10 + трейлинг последней части; `[069c4600]` «была система: 1к3 → закрываю 50%, переношу в безубыток»; `[503ef59e]` «БУ вводить ТОЛЬКО после статистики». → включать опционально, не дефолтом.

**How to apply (commit e763bd4):** `paper_engine.BE_AT_R=0` (выкл), `eng.be_at_r` из config `be_at_r` (дефолт 0). Лесенка (§9 п.4) и трейл — НЕ внедрять как база; только опционально после набора статистики. ⚠ Своды `course_findings_80` подавали лесенку/БУ как метод — это переакцентировка экстракторов; база проще.
