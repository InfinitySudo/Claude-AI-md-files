2026-06-13: на панели отбой/пробой = 0 сделок, у ЛП — позиция. Диагностика частоты срабатывания
триггеров по истории (30 монет, 10248 моментов «цена у уровня ±0.5%»):
- **ЛП: 99%** (любой недавний прокол хвостом = триггер; селективность в поздних гейтах, не в триггере)
- **пробой: 5.2%** (нужен последний дневной close за near-уровнем + БПУ) — редко, но живой; 0 пока = мало времени
- **отбой: 0.13%** (13/10248) — практически НИКОГДА

**Корень отбоя:** `volume_levels.level_not_pierced` требовал, чтобы уровень не был пробит ТЕЛОМ
**ни разу за всю историю (200 D)**. В волатильной крипте почти каждый уровень когда-то пройден →
~99.9% уровней дисквалифицированы → отбою не от чего отбиваться.

**Fix:** `level_not_pierced(level, klines, lookback)` + `strategies.OTBOI_HELD_LOOKBACK` (config
`otboi_held_lookback`). Окно = «уровень держится на ТЕКУЩЕМ подходе» (последние N D-баров), а не за всю
историю — это и вернее методу (для отбоя важна свежая структура). Замер: lookback 0→60 поднимает
срабатывания **13→200 (×15, 1.95%)** — отбой ожил, но остался селективным (не залит как ЛП).
Живьём `otboi_held_lookback=60`. Замеряем, начнёт ли отбой давать сделки.

**Урок:** правило Герчика верное для акций, но порог/окно надо калибровать под волатильность крипты
(ср. [[feedback_gerchik_turnover_partial_day]], [[feedback_gerchik_window_dual_role_limit]]).
пробой не трогаем — просто ждём накопления. См. [[project_gerchik_instrument_selection]].
