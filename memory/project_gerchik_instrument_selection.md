2026-06-12: извлёк из курса (урок b95db6ef «практичный отбор инструментов») по каким параметрам
А.М.Герчик выбирает что торговать, и применил к нашему агенту (PumpDumpAI_Agent, gerchik.json).

**Критерии Герчика (по порядку важности):**
1. ЛИКВИДНОСТЬ (первичный фильтр) — оборот за 24ч в $ ≥ 50М (ползунок в TradingView-скринере,
   часто 50–100М → ~62 инструмента из ~423). «В первую очередь по ликвидности, чтобы могли торговать».
2. КАЧЕСТВО ОБЪЁМА (анти-памп) — смотреть КАК набран объём; мизерный историч. объём, резко
   «пухнущий» = памп → удаляет «на этапе давления». Откуда объём, постоянный ли, не разовый всплеск.
3. ЭНЕРГИЯ/ХОД — «если инструмент не имеет хода, его тяжело торговать» → отсев слабых/отыгравших.
4. ЧИТАЕМОСТЬ — ликвидный, но хаотичные движения / много длинных хвостов / свежие монетки →
   «сразу удаляю, посадят на хвост, не суперчитаем».

**Что было у нас:** `selection.is_tradeable` уже делал ликвидность (`turnover_usd`≥50М, среднее полных
дней — см. [[feedback_gerchik_turnover_partial_day]]) + энергию (ATR≥1.5%) + относит. силу vs BTC.
**Добавил (selection.py + config-ключи в gerchik.json, подхват в run_gerchik):** свежесть
`min_history_days=60`, анти-памп `max_turnover_spike=3.0` (оборот_последнего_дня/медиана30), читаемость
`max_wick_ratio=0.70` (доля хвостов в баре). Диагностика: `scripts/gerchik_screen.py`.

**⚠ ГЛАВНЫЙ ВЫВОД — порог $50M венчурно-строгий для Bybit.** Герчик мерит оборот на TradingView
(агрегат/Binance), а у нас оборот ТОЛЬКО Bybit-linear (доля от总). При 50М on-Bybit отсеиваются
РЕАЛЬНО ликвидные мейджоры: BNB(37M) LINK(46M) BCH(30M) AVAX(34M) LTC(27M) ENA(44M) — Герчик их бы
торговал. Из 33 живых проходят только 14. Рекомендация: снизить `min_turnover_usd` до ~25–30M под
реальность Bybit (тогда вернутся мейджоры, отсекутся только тонкие HOME6M/OP13M/TRX12M/ALLO15M/APT16M).
Решение по числу — за Артёмом (размер вселенной/риск). Гарды свежесть/памп/хвосты сейчас не режут
живой набор (памп-джанк HOME/HU/BEAT уже отсечён 50М-фильтром), но корректны на будущее.
