2026-06-13: «как агент выбирает вид сделки». БЫЛО: 3 НЕЗАВИСИМЫХ движка (отбой/пробой/ЛП) гоняются
параллельно на каждом символе, каждый входит если сработал ЕГО триггер; арбитра/селектора стиля НЕТ
(курс-aligned: статистику по трём собирают отдельно [c0560563]). «Выбор» был неявный = геометрия
цена↔уровень в триггерах. Минус: в тренде стили конфликтуют (ЛП-шорт фейдил растущий TAO, хотя верный
был пробой-лонг) — контекст-сигналы near/far_retest были только EDGE-бонусом, не гейтом.

**Правило курса [eee2e992][a9ae6c35]:** ближний ретест+поджатие+энергия+начало ATR → ПРОБОЙ;
дальний ретест+длинный хвост+исчерпание → ОТБОЙ/ЛП.

**Сделано (pipeline.py build_setup, config-тумблеры, run_gerchik):** гейты выбора стиля —
- пробой требует `near_retest` (`style_proboi_near`), skip `proboi_far_retest`;
- отбой/ЛП требуют `far_retest` (`style_fade_far`), skip `fade_near_retest`;
- ⚠ `style_proboi_atr_room=FALSE` — atr_room НЕСОВМЕСТИМ с детектором пробоя (вход на закрытии за
  уровень → ход уже пройден → atr_room=0% → зануляет пробой; замер показал 538→1).

**Замер на истории (30 монет, кэш /tmp/gerchik_bt_full):** триггер→после гейтов:
пробой 538→237 (44%, near), отбой 200→169 (84%, far), ЛП 10130→5549 (55%, far) — все живы, дисциплины
больше (ЛП фирил 99%, теперь только дальние ретесты). Живьём ВКЛ; пробой реально торгует (2 позиции).
Дополняет фейд-гейт [[feedback_gerchik_judge_oauth_429_fade_gate]]. Скрипты замера ad-hoc (не в репо).
