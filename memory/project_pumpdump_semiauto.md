2026-06-08: PumpDumpAI ([[project_pumpdump_agent]], [[project_pumpdump_selftuning]]) переведён на **полуавтоматическую торговлю** через дашборд `/var/www/dashboard/pumpdump.html` (зеркало Space_Live/public; работает ТОЛЬКО на `:8080`, НЕ на `:443`/ontime — там `/pumpdump/*` кроме webhook не проксируется). Всё PAPER (sub-4).

**Решение:** старый WS-детектор (vol×5+3%) ОТКЛЮЧЁН от авто-входа (`detection.auto_enabled=false`) — давал 95% SL (TP=5R недостижим, median MFE 0.8%; only 2 TP3 на 559 сделок). Детектор крутится только для live-цен/статистики. Торговля = reversal-подсказки + ручные.

**Что реализовано (всё за этот день, много коммитов):**
- **График** TradingView Lightweight Charts (self-hosted `lightweight-charts.js`): клик по символу → свечи+объём+маркеры входов/выходов (цвет по source) + тултип с rationale; инструменты (fullscreen ВСЕЙ секции, гор.линии, **линейка** drag с Δ%/Δ/бары, EMA20/50); живое авто-обновление 5с (setData без fitContent); TF 1m/5m/15m/30m/1h/2h/4h/1D; авто-rescale при смене монеты; precision под мелкие цены.
- **Ордер-тикет** (📈/📉): Market (налив по live) / Limit (очередь pending, исполняется по достижению цены, /pending-cancel); размер от risk/margin/notional/qty; редактируемые SL/TP/BE; live qty/notional/margin/risk + прибыль при TP / убыток при SL / R:R; кнопка «SL=5м свеча» (медиана 12 не-аномальных 5м-свечей, аномалии <0.5× / >1.8× медианы). Floor **TP1 ≥ 3R** (`tp.tp1_min_R`).
- **Подсказки Take/Skip** (suggest-режим сканера): карточки с чек-листом критериев + rationale; Take открывает source=suggested.
- **Перетаскиваемые SL/TP на графике** (ByBit-style): drag линий открытой позиции, live −$убыток/+$прибыль; commit → `/position-update` (PAPER лог / REAL пушит на Bybit через executor.set_sl_tp).
- **Панель «Открытые позиции»** — live нереализованный PnL ($/R/%), объём, плечо, до SL/TP.
- **Скриншот-канал** (см. [[project_pumpdump_selftuning]]): `data/screenshots/latest.png`.

**Эндпоинты агента (http_server):** /klines, /symbol-trades, /symbols, /manual-trade, /suggestion-act, /pending-cancel, /position-update, /screenshot, /tune-now, /rollback-tuning (+ /stats несёт positions, pending_orders, suggestions, recent_signals с исходом). trade_open несёт `source` (detector/signal/suggested/manual/limit) + `rationale`.

**Анти-лаг/анти-падение (2026-06-08):** симптом «OFFLINE :8004» = не падение процесса, а залип event-loop (сервис active, /stats 200, но /klines висит 12с→502; хост достаёт Bybit за 47мс). Причины+фиксы: (1) `/stats` парсил весь журнал дважды каждые 5с → `journal.read_all` кеш по (mtime,size); (2) klines/orderbook поднимали новый `ClientSession` на запрос → общий session на app (on_startup/on_cleanup, TCPConnector pool); (3) шарды: Артём снизил `max_topics_per_ws` 400→100 (ws в agent.json), CPU 90%→11%; (4) **WS orderbook.50** — новый `src/orderbook_ws.py` (snapshot+delta, ленивая подписка по touch(), idle-evict 30с, REST-фолбэк на первый кадр); `/orderbook` отдаёт `source:ws|rest`; (5) systemd: `Restart=always`/`RestartSec=5`, `MemoryMax 512→896M`+`MemoryHigh 640M`, **watchdog** `pumpdump-health.timer` (60с) дёргает `scripts/healthcheck.sh` → рестарт после 2 провалов /klines. Бэкапы: `pumpdump.service.bak-20260608`. Нагрузка 120 паралл. = все 200, ≤0.11с.

**Усреднение позиций (2026-06-08):** `Tracker.add_position` раньше складывал позиции по символу списком и `.append()` → повторный ручной вход в тот же символ+сторону плодил ДУБЛЬ, а не усреднял (Артём словил 2×BIO LONG, 3×VELVET SHORT). Фикс: `add_position` сливает долив в открытую позицию того же symbol+side через `_average_into` (суммарный qty, взвешенная средняя entry, SL/TP пересчитываются от новой средней по qty-взвешенной стоп-дистанции + R-кратностям свежего тикета; entry-fee лениво от финальных entry×qty = верный ноционал). НЕ усредняет если уже были TP-филлы (держит раздельно + warning). Mode-agnostic: REAL Bybit усредняет нативно (one-way), а локальный сингл-на-символ нужен чтобы watchdog-реконсайлер (`remote_syms` по символу) не путался. Слил текущие 2×BIO одноразовым скриптом (data/open_positions.json.bak-20260608). ⚠ TP-лесенка может быть немонотонной — это клампинг `tp1_min_R=3` в тикете, не баг усреднения.

**TP/SL не срабатывал по открытой позиции (2026-06-08):** в PAPER `tracker.on_price` (драйвер TP/SL-филлов, локальный учёт без биржевых ордеров) кормился ТОЛЬКО WS-тиками детектора; watchdog REAL-only. По BIO тики были редкими → цена прошла TP3 (0.03315), а трекер тика не увидел → TP не отработал, линия осталась. Фикс: `_position_price_loop` в main.py — каждые 3с REST `lastPrice` по символам открытых позиций → `on_price` (mode-agnostic; в REAL держит локальный учёт в синхроне с биржей). BIO TP3 зафиксировался сам после рестарта (цена ещё была выше уровня). ⚠ peak/last_price персистятся на диск только при структурных изменениях (TP/BE/close), в памяти обновляются каждый тик — это by design. ⚠ Немонотонная TP-лесенка (TP1 на 3R клампе дальше TP2/TP3) сбивает с толку: «TP3» по списку может филлиться раньше «TP1».

**Day PnL (2026-06-08):** был in-memory в RiskManager → обнулялся при каждом рестарте + копил только полные закрытия. Фикс: считается из журнала (`stats_24h` → `day_pnl_usd`, сумма net с полуночи UTC), http_server отдаёт журнальный, не rm-snapshot.

**Скриншот-снип в fullscreen (2026-06-08):** Fullscreen API рендерит только top-layer элемент; снип-оверлей/превью висели на `document.body` → под слоем (невидимы), html2canvas снимал не тот слой. Фикс в pumpdump.html: оверлей/превью монтируются в `document.fullscreenElement||body`, captureRegion снимает fullscreen-элемент с пересчётом координат. Кнопки тулбара внутри `#chart-section` (кликаются в fullscreen). Бэкап `pumpdump.html.bak-20260608c`.

**Риск-сетка на графике (2026-06-08):** новый инструмент `📐 риск-сетка` в pumpdump.html — штрихпунктирные линии с шагом 1R (R=SL в цене). Построение: одиночный клик = бесконечная сетка ±R от уровня; протяжка = диапазон (показывает «сколько R»); 2 точки на разной цене/времени = авто-наклон (для трендов/каналов). Источник 1R — переключатель `risk-src`: «12 свечей (5м)» (как кнопка SL=5м, медиана-фильтр аномалий, всегда по 5м → TF-независимо) ИЛИ «настройки %» (`sl_pct_min`). Сетка в ценовых единицах → работает на всех TF; сбрасывается при смене монеты. Реализация: SVG-оверлей `chart-ovl`, пиксельная математика (pxPerR), raw-mouse клик/протяжка. Бэкап `pumpdump.html.bak-20260608d`. Протяжка теперь = **ограниченный box** (сетка только внутри выделенного прямоугольника время×цена, наклон базовой линии сохраняется, рамка зоны + «N.NR» итог); одиночный клик = бесконечная ±R на весь экран. Скриншот-канал 413: nginx `/pumpdump/` дефолтный лимит 1MB резал PNG → `client_max_body_size 30m` в sites-available/dashboard (символ! правил реальный файл). 🎯 ПЛАН (Артём, обучающий курс пришлёт): AI-агент торгует в диапазоне заданном сеткой/линиями — отбои от границ канала, пробои, ложные пробои; решение по предпосылкам + поведению баров. Пока полуавтомат.

**Рисовалки переживают ТФ/монету/перезагрузку (2026-06-08, commit 7d2813f):** `loadChart()` безусловно чистил `_rLevels`/`_hlines`/`_measures`/`_shapes`/`_riskGrid` (комментарий «смена символа»), но вызывается и при смене ТФ → R-диапазон и линии исчезали при каждом переключении таймфрейма. Фикс 1: флаг `_symChanged=(sym!==_chartSym)` — очистка только при реальной смене монеты (цены TF-независимы). Фикс 2: персист per-symbol в localStorage `pd_draw_<SYMBOL>` (`_saveDraw`/`_restoreDraw`, ключи r/h/m/s/g/seq; hlines пересоздают priceLine при восстановлении, shapes/measures/riskGrid рисуются по t1/t2 через timeToCoordinate). `_saveDraw` вызывается из `_renderToolPanel()` (единая точка коммита всех мутаций); guard `_drawLoading` подавляет сохранение во время restore, чтобы пустота не затёрла данные при смене монеты. Очистка/удаление → ключ удаляется. Источник правды pumpdump.html = Space_Live/public, зеркалится в /var/www/dashboard. Round-trip проверен node-тестом.
