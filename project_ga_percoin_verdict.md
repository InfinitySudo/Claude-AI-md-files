**Решающий тест (2026-06-06):** взяли конфиг-победитель GA Trader E (regime ER≥0.538, окно 2026-01-08..06-07, 197 монет) и померили net ПО КАЖДОЙ монете на test-окне (train_split 0.70). Скрипт `/tmp/per_coin_eval.py` → PC1 (тот же движок `evaluate_strategy`, per-trade несёт sym_idx). Воспроизвёл отчёт GA байт-в-байт: test net −21.15 / gross +26.68 / 149 трейдов, train +17.39 / 439.

**Результат:**
- 77 монет с тестовыми сделками; **60/77 = 77.9% net-плюсовые**, медиана per-coin net **+0.13** → эдж НЕ чистый averaging-мусор, режим-фильтр ловит реальную широкую тенденцию.
- НО суммарно −21.15, потому что **хвост убытков**: выигрышные 60 монет = +35.39, убыточные 17 = −56.54; топ-5 убыточных = −32.48 (RAVE −11.6, ZEC −7.5, PEPE, TON, INJ). Убрать топ-5 → basket +11.33.
- Caveat в сторону скепсиса Артёма: среди монет с **≥3 сделок** (меньше шума) только **7/17 плюсовые** → headline 77.9% раздут одно-двухсделочными монетами (везучий единичный вход).

**Вывод:** GA работает и НЕ хлам (median+, 78% монет+), но и НЕ даёт деплоебельный эдж. Узкое место — **не оптимизатор, а риск-модель**: SL ATR×0.37 слишком широкий на волатильных альтах, отдельные сделки теряют куда больше ~1.5R (RAVE −2.9 net/сделку). Полировка fitness GA это не чинит.

**Реальный рычаг (не «больше поколений»):** ~~(1) кап на убыток per-trade~~ — ПРОВЕРЕНО, НЕ РАБОТАЕТ (см. ниже); (2) min N сделок на монету прежде чем доверять; (3) дроп fat-tail монет только при структурной причине (ликвидность/слиппедж), иначе кёрв-фиттинг. Workflow: довести робастный конфиг на PC1 → отдать на PC2 (идентичный код). См. [[project_ga_regime_feature]], [[feedback_pc2_cupy_ctk]], [[project_trader_model_10]].

**ОБНОВЛЕНИЕ 2026-06-06 #2 — диагностика КАЧЕСТВА ВХОДА → найдена селективность как реальный рычаг.** Диаг конфига E (test):
- #1 forward-return (signal-aligned): +6б −0.11%, +12б −0.15%, ~0/слегка минус → сигнал НЕ предсказывает направление (adverse selection: вход в спайк = погоня).
- #2 инверсия направления: по сигналу −21.15 vs против −19.56 → симметрично, направление НЕ backwards, эджа в направлении нет.
- ⚠ Данные prefetch = OHLCV без buy/sell; kernel B/S = **60/40 прокси по направлению бара**, НЕ настоящий order-flow. Реальный aggressor-объём не тестируем (Bybit kline не отдаёт; нужен trades/aggTrades). Backtest B/S ≠ live B/S.
